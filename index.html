API Reference
Base URL
https://web-production-512d4.up.railway.app
POST /identities
Create an identity. No auth required. Returns a token — store it.
curl -X POST https://web-production-512d4.up.railway.app/identities // 201 response { "token": "eyJhbGciOiJIUzI1NiJ9...", // your key — keep it "name": "Sophie Baker", "email": "sophie.baker@inbox.droid.dev", "phone": "+1 361 322 2054" }
GET /inbox
Read all incoming emails for this identity.
curl https://web-production-512d4.up.railway.app/inbox \ -H "Authorization: Bearer <token>" // 200 response { "count": 1, "emails": [{ "from": "noreply@twitter.com", "subject": "Confirm your email", "body": "Your code is 482910. Click here...", "received": "2026-03-25T14:32:10Z" }] }
GET /sms
Read all incoming SMS messages for this identity.
curl https://web-production-512d4.up.railway.app/sms \ -H "Authorization: Bearer <token>" // 200 response { "count": 1, "sms": [{ "from": "+15005550006", "body": "Your LinkedIn code: 739201", "received": "2026-03-25T14:33:44Z" }] }
Authentication
TypeJWT Bearer token — pass in Authorization header
ScopeOne token = one identity. It encodes the identity ID inside.
Expiry1 year from creation
401Missing or invalid token
500Server error — retry or contact arthur@droid.dev
Autonomous usage pattern
1.POST /identities → save token, name, email, phone
2.Use name / email / phone to sign up on the target service
3.Poll GET /inbox every 5s — parse body for codes or links
4.If SMS expected: poll GET /sms — same logic
5.Act on whatever you find. Full raw content is returned.
